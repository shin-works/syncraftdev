# Blog Policy — SynCraft コンテンツ制作指針

## 目的

SEO および LLMO（LLM Optimization）対策のためのコンテンツ制作指針。
検索エンジンと AI モデルの両方に引用されやすい構造的なコンテンツを制作する。

## 記事ルール

### 基本

- **1 記事 1 キーワード**: メインキーワードを 1 つ決め、タイトル・H2・本文に自然に含める
- **英語版は必ず作成**: 日本語と英語は翻訳ではなく **ローカライズ**（文化・文脈を調整）
- **FAQ を必ず含める**: 記事末尾に最低 3 つの Q&A を配置
- **H2 を最低 3 つ**: 記事の構造を明確にし、スキャンしやすくする
- **内部リンク必須**: 対象プロダクトページへのリンクを必ず含める

### URL / Slug

- スラッグは **短く、キーワードを含める**
- 抽象的なワード（"updates-and-insights" など）は禁止
- 英語ページは英語 slug のみ
- 各言語独立 slug 可（日英で異なる slug OK）

### 構成テンプレート

```
H1: [タイトル — メインキーワードを含む]
Lead文: 1-2文で記事の要約

H2: [トピック1]
本文...

H2: [トピック2]
本文...

H2: [トピック3 / 実践的なアドバイス]
本文...

H2: FAQ
Q&A × 3以上

[プロダクトCTA]
[関連記事リンク]
```

## LLM 対策（LLMO）

AI モデルに引用されやすいコンテンツを作るための指針。

### 定義文を含める

- 記事の冒頭付近に、メイントピックの **定義文** を 1-2 文で記載
- 例: 「ビジュアルタイマーとは、残り時間を色や形で視覚的に表示するタイマーのことです。」

### 箇条書きを活用

- 特徴、メリット、手順などは **箇条書き** で整理
- LLM は構造化された情報を引用しやすい

### 問題 → 解決構造

- 読者の課題を明示し、それに対する解決策を提示する構造
- 例: 「朝の準備が毎日バタバタ → TimeSense で声かけタイマーを設定」

### 専門用語の説明

- 専門用語は初出時に **括弧書き** で説明
- 例: 「ADHD（注意欠如・多動症）の子どもにとって…」

### 構造化データ

- 各記事に **Article JSON-LD** を自動挿入（Astro の SEOHead コンポーネントで対応済み）
- FAQ がある場合は **FAQPage JSON-LD** の追加も検討

## SEO チェックリスト（記事公開前）

- [ ] タイトルにメインキーワードが含まれている
- [ ] meta description が 120-160 文字で設定されている
- [ ] H1 はページに 1 つだけ
- [ ] H2 が最低 3 つある
- [ ] FAQ セクションがある（最低 3 問）
- [ ] プロダクトページへの内部リンクがある
- [ ] 英語版（ローカライズ版）が作成されている
- [ ] frontmatter の `alternates` で対応言語の slug が設定されている
- [ ] 画像に alt テキストが設定されている
- [ ] slug が短く、キーワードを含んでいる

---

## 運用フロー

1. キーワードリサーチ → メインキーワードを 1 つ決定
2. 日本語版を `src/content/blog/ja/{slug}.md` に作成
3. 英語版を `src/content/blog/en/{slug}.md` にローカライズ作成
4. frontmatter の `alternates` を相互設定
5. `npm run build` でビルド確認
6. SEO チェックリストを確認
7. デプロイ
